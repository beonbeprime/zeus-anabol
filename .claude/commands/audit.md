# Auditoria Completa do Projeto

Faca uma auditoria completa deste repositorio e entregue um relatorio estruturado.

## O que analisar

1. Estrutura geral
   - Quais arquivos e pastas existem
   - Stack tecnologica identificada
   - Padroes de organizacao

2. Qualidade do codigo
   - Inconsistencias de padrao
   - Codigo duplicado ou morto
   - Complexidade desnecessaria
   - Falta de tratamento de erros

3. Seguranca
   - Dados sensiveis expostos
   - Inputs sem validacao
   - Dependencias com vulnerabilidades conhecidas
   - Permissoes excessivas

4. Divida tecnica
   - O que esta funcionando mas precisa de refatoracao
   - O que esta incompleto
   - O que pode quebrar com crescimento

5. Quick wins
   - Melhorias de alto impacto e baixo esforco
   - O que resolver primeiro

## Formato de saida

Use este formato:

### SITUACAO ATUAL
[resumo em 3-5 linhas]

### PROBLEMAS CRITICOS
[lista com arquivo:linha quando aplicavel]

### DIVIDA TECNICA
[lista com impacto estimado]

### SEGURANCA
[lista ou "nenhum problema identificado"]

### QUICK WINS
[lista ordenada por impacto]

### PLANO RECOMENDADO
[etapas numeradas com prioridade]

Seja direto. Nao omita problemas para soar gentil.
Use o agente researcher para investigar partes especificas se necessario.
